Projects

Sudoku

Motivation of the project
My partners and I share a deep appreciation for puzzle games and in light of the growing popularity of games like Wordle & Crosswords, we immediately knew Sudoku would be our project. The game isn't too simple like Wordle and it's not subjectively complex like Crosswords, thus, this very much appealed to our sweet spot. As if playing Sudoku wasn't intriguing enough, developing the puzzle from scratch was too large an opportunity to pass up on.

Functional requirements/what the project does
This program offers a custom Sudoku experience on a classic 9x9 Sudoku board. Players must solve the puzzle by entering digits 1 through 9 into their appropriate cells, ensuring that each digit only appears once per row, column, and 3x3 block. In other words, each row, column, and block will contain exactly one occurence of digits 1 through 9. Players can select from three levels of difficulty: easy, medium, and hard. Upon choosing a difficulty level, a board is randomly selected from a large collection of Sudoku boards, and then the game begins. Successfully completing the puzzle will congratulate the player and prompt them to play another round.

Project design
We decided to make our Sudoku board interactive through a GUI. For the GUI, we used Java Swing which allowed us to display the board in a 9x9 text-box format. This interface displays the game title, difficulty levels, the Sudoku Board, and messages toward the player. If this program interface were to be implemented through the terminal, it would heavily complicate the simplicity of the game for the player. With our GUI, the user can enter the value in the cell they'd like to check as a valid input.

Implementation
I played a key role in implementing the core of our Sudoku board. Specifically, I was responsible for a significant portion of the following classes: Board, BoardInterface, Row, Column, Block, Cell. Planning out these classes, their methods, and their instance variables in the design.md file made this process very smooth. Also, when logic differed between Row, Column, and Block, paired-programming proved to be very helpful. We could exchange immediate feedback resolve issues more quickly than if I were coding by myself.
